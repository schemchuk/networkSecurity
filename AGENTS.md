# AGENTS.md — Інструкції для Kimi

Кодогенерацію в цьому проекті виконує Kimi за специфікаціями від Claude.
Повний цикл — у [docs/specs/README.md](docs/specs/README.md).

## Старт задачі
1. Відкрий призначений спец `docs/specs/Mx.y-*.md`.
2. Прочитай лише його + файли, вказані в ньому. Дотримуйся розділу «НЕ чіпати».
3. Якщо спец незрозумілий → спитай user, не починай.

## Виконання
- Реалізуй рівно те, що в спеці. Python, строга типізація, docstring-и.
- Не додавай залежностей понад перелічені у спеці без потреби.
- Перед поверненням прожени всі команди з розділу «Acceptance criteria»
  (активуй `.venv`). Усі мають бути зелені. Тести не ходять у мережу.
- **НЕ коміть і НЕ пуш** — це робить Claude після рев'ю diff-у.

## Формат звіту (обов'язковий, після завершення блоку)

Звітуй **коротко**, рівно три частини:

1. **Змінені файли** — список створених / оновлених файлів.
2. **Acceptance** — команди перевірки та їх результат із числами
   (напр. `pytest -q → 12 passed`; `bash -n scripts/labshell → OK`).
3. **Відхилення / рішення** — усе, що зроблено інакше, ніж у спеці, нетривіальні
   рішення, обрані підходи чи бібліотеки. Якщо нема — напиши «відхилень нема».

**НЕ переказуй логіку коду** — Claude читає diff. Розділ 3 — найважливіший:
саме він допомагає швидко знайти ризиковані місця.
