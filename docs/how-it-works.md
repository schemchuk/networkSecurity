# Як це працює — розповідь про проект з нуля

> Для того, хто «щойно зліз з дерева»: без жаргону, з аналогіями, з картою «де що лежить».
> Читай згори вниз — кожен розділ спирається на попередній.

---

## 1. Що це взагалі таке (в трьох реченнях)

Ти будуєш **власну навчальну лабораторію з пентесту** (перевірки систем на вразливості) і
**команду локальних AI-помічників**, які автоматично аналізують те, що ти в цій лабораторії
знаходиш.

Уяви заводський **конвеєр**: з одного боку заїжджає сирий результат сканування мережі, а з
іншого виїжджає готовий, охайний **звіт** — які сервіси відкриті, які в них відомі діри
(CVE), що найнебезпечніше, як це закрити і як помітити атаку.

Усе працює **локально й офлайн** — жодних хмарних сервісів, ніщо не йде в інтернет.

---

## 2. Навіщо це (мета)

Дві цілі одночасно:
1. **Навчитися** пентесту руками — на безпечному стенді, а не на чужих системах.
2. **Зробити портфоліо** — реальний проект, який можна показати роботодавцю зі словами:
   «я побудував лабораторію + AI-агентів, які з сирого сканування роблять готовий звіт».

> ⚠️ Усе — **тільки** у власній ізольованій лабораторії. Правила — в
> [SCOPE.md](SCOPE.md). Це не інструмент для атак на чужі системи.

---

## 3. Базові поняття простими словами

Щоб далі все було зрозуміло — маленький словник (докладніший є в кінці, розділ 9):

- **Пентест (penetration testing)** — легальна перевірка «а чи можна сюди залізти?», щоб
  знайти й полагодити діри раніше за зловмисника.
- **Лабораторія (lab)** — кілька віртуальних машин на твоєму комп'ютері, з'єднаних в
  окрему мережу без виходу в інтернет. Одна машина «атакує» (Kali), інші — «жертви»
  (навмисно вразлива **Metasploitable2**, Windows).
- **Ланцюг пентесту:** `recon → vuln analysis → exploitation → hardening → detection`
  (розвідка → аналіз вразливостей → експлуатація → захист → виявлення). Ми
  автоматизуємо **аналітичні** етапи (розвідка, вразливості, захист), а не злам.
- **nmap** — сканер мережі. Питає в машини-жертви «які в тебе відкриті двері (порти) і
  що за ними працює?» і видає відповідь.
- **Порт / сервіс** — «двері» й програма за ними (напр. порт 21 = FTP-сервер vsftpd 2.3.4).
- **CVE** — світовий каталог відомих вразливостей. `CVE-2011-2523` — конкретна діра в
  конкретній програмі. Кожна має номер.
- **LLM (велика мовна модель)** — «розумний текстовий двигун» (у нас — `qwen2.5:3b`, що
  крутиться локально через програму **Ollama**). Вміє прочитати факти й написати
  осмислений текст: резюме, поради.
- **Агент** — маленька програма-працівник з однією роллю. У нас їх кілька, кожен робить
  свій крок конвеєра.

## 4. Що система робить — крок за кроком

Уяви **бригаду з чотирьох спеціалістів**, які передають одну теку одне одному:

```
 nmap-скан (сирий XML)
        │
        ▼
 [1] Парсер ── читає XML, витягує список: хост, порт, сервіс, версія
        │
        ▼
 [2] Recon-агент (LLM) ── до кожного сервісу пише коротке резюме:
        │                  що це за поверхня атаки, куди дивитись далі
        ▼
 [3] Vuln-агент ── шукає РЕАЛЬНІ CVE для кожної версії (у локальній базі
        │           exploitdb) і розставляє пріоритети: що найнебезпечніше
        ▼
 [4] Hardening-агент (LLM) ── для найважливіших знахідок пише:
        │                      як закрити (mitigation) + як помітити (detection)
        ▼
 Готовий звіт report.md  (охайний Markdown: таблиця + деталі)
```

Живий приклад того, що виходить — [portfolio/sample-report-metasploitable.md](portfolio/sample-report-metasploitable.md).
Це справжній звіт з реального прогону проти Metasploitable2.

## 5. Одна головна ідея, яку варто зрозуміти

**Точну роботу робить звичайний код, а не AI.**

- Витягти порти з XML, знайти CVE у базі — це **детерміновані** кроки (звичайний Python).
  Вони завжди дають однаковий точний результат.
- AI (LLM) використовується **тільки** там, де треба написати текст людською мовою:
  резюме, поради із захисту.

Чому так? Бо LLM **може вигадувати** (галюцинувати). Якби ми просили модель «назви CVE» —
вона могла б вигадати номер, якого не існує. Тому CVE ми **знаходимо в реальній базі**, а
модель лише пояснює. Це і надійніше, і сильний аргумент на співбесіді.

## 7. Як це запустити і що ти побачиш

Крок за кроком, у терміналі на твоєму головному Kali:

```bash
# 0. зайти в проект і активувати робоче середовище
cd ~/VS_Code_projects/networkSecurity
source .venv/bin/activate

# 1. увімкнути машину-жертву (Metasploitable2) у VirtualBox
VBoxManage startvm metasploitable2 --type headless
#    (зачекай ~1 хв, поки завантажиться; перевір: ping -c2 192.168.56.101)

# 2. переконатися, що AI-двигун працює (зазвичай уже так)
#    в окремому терміналі: ollama serve

# 3. просканувати жертву nmap-ом → отримати сирий XML
nmap -sV -oX /tmp/scan.xml 192.168.56.101

# 4. НАТИСНУТИ ГОЛОВНУ КНОПКУ — запустити весь конвеєр
python scripts/run.py --nmap /tmp/scan.xml --scenario мій-прогін --targets 192.168.56.101
#    → надрукує шлях, напр.: runs/2026-.._мій-прогін/report.md

# 5. подивитися звіт
cat runs/2026-.._мій-прогін/report.md      # або відкрити цей файл у VS Code

# 6. вимкнути жертву, коли закінчив
VBoxManage controlvm metasploitable2 poweroff
```

Що відбувається під час кроку 4 (це може тривати ~10-15 хв на твоєму залізі): бригада
агентів по черзі обробляє кожен порт, модель пише резюме й поради, система шукає CVE — і
складає все у звіт.

## 8. Що вже зроблено (простими словами)

Проект будувався «поверхами» (milestones):
- **M0** — фундамент: спільні форми даних, зв'язок з моделлю, журналювання.
- **M1** — перший робочий конвеєр: скан → резюме → звіт.
- **M2** — додали **реальні CVE** й пріоритети.
- **M3** — додали **поради із захисту** (mitigation/detection).
- **M5** — упакували в портфоліо (цей текст, README, приклад звіту, бенчмарк).

**Відкладено до нового ноутбука** (бо слабке залізо): RAG (розумна база знань) і LangGraph
(складніша оркестрація). Чому — у [hardware-upgrade-rationale.md](hardware-upgrade-rationale.md).

## 9. Словничок (якщо десь застряг)

| Термін | Простими словами |
|---|---|
| **Kali** | ОС для пентесту; твоя головна робоча машина |
| **VirtualBox** | програма, що крутить віртуальні комп'ютери (VM) всередині твого |
| **Metasploitable2** | навчальна VM, навмисно повна дір — безпечна ціль для тренувань |
| **порт** | «двері» на машині; за кожними працює якийсь сервіс |
| **сервіс** | програма за портом (FTP, SSH, веб-сервер…) |
| **nmap** | сканер: показує відкриті порти й що за ними |
| **CVE** | номер відомої вразливості у світовому каталозі |
| **exploitdb / searchsploit** | локальна база експлойтів; звідти беремо реальні CVE |
| **LLM** | AI, що працює з текстом (у нас `qwen2.5:3b`) |
| **Ollama** | програма, що запускає LLM локально на твоєму комп'ютері |
| **агент** | маленька програма-працівник з однією роллю |
| **pipeline (конвеєр)** | послідовність агентів, що передають роботу далі |
| **finding (знахідка)** | один запис: цей порт, цей сервіс, ці CVE, ця порада |
| **mitigation** | як усунути/зменшити вразливість |
| **detection** | як помітити, що хтось намагається її використати |
| **Markdown (.md)** | простий формат тексту з заголовками й таблицями (як цей файл) |
