# M0.1 — Python-скелет + схеми даних + конфіг

**Контекст:** реалізуємо фундамент з `docs/ARCHITECTURE.md`. Це блок 1 з M0.
Схеми даних — точно за §7 архітектури (finding / event / manifest).

## Ціль
Робочий Python-пакет проекту з валідованими Pydantic-схемами та лоадером конфігу.
Без бізнес-логіки агентів — лише каркас, на який усе спиратиметься.

## Залежності (додати у `requirements.txt` + `pyproject.toml`)
- `pydantic>=2`
- `pyyaml`
- (dev) `pytest`

## Файли, які треба створити

### 1. Пакет і збірка
- `pyproject.toml` — назва проекту `ai-cybersec-lab`, Python `>=3.10`, пакети `tools`, `agents`.
  Editable-install (`pip install -e .`) має працювати.
- `tools/__init__.py`, `agents/__init__.py` — зробити їх пакетами.
- `requirements.txt` — залежності вище.

### 2. `tools/schemas.py` — Pydantic-моделі (за §7 ARCHITECTURE.md)
- `CVE`: `id: str`, `cvss: float | None = None`, `source: str`.
- `Severity`: Enum `info|low|medium|high|critical`.
- `Finding`: поля `id, host, port(int|None), protocol(str|None), service(str|None),
  product(str|None), version(str|None), source(str), cves: list[CVE] = [],
  severity: Severity = info, priority: int | None = None, summary: str = "",
  mitigation: str = "", detection: str = "", next_steps: list[str] = [],
  evidence: str | None = None`.
- `Event`: `ts: datetime`, `run_id: str`, `agent: str`,
  `type: Literal["finding_added","llm_call","error","note"]`, `ref: str | None = None`,
  `data: dict = {}`. Додати `@classmethod now(...)` для зручного створення з поточним UTC-часом.
- `Manifest`: `run_id: str`, `started: datetime`, `scenario: str`, `targets: list[str]`,
  `operator: str`, `tool_versions: dict[str,str] = {}`.
- Усі моделі мають серіалізуватись у JSON (`model_dump_json`) і назад.

### 3. `configs/model.yaml` — конфіг LLM (hardware-agnostic)
```yaml
backend: ollama
host: "http://localhost:11434"
model: "qwen2.5-coder:7b"
embed_model: "nomic-embed-text"
context_tokens: 8192
temperature: 0.2
```

### 4. `tools/config.py` — лоадер конфігу
- `class ModelConfig(BaseModel)` з полями під `model.yaml`.
- `load_model_config(path: str = "configs/model.yaml") -> ModelConfig` — читає YAML, валідує.
- Дозволити override шляху через env `AILAB_MODEL_CONFIG`.

## Acceptance criteria (Kimi має прогнати перед поверненням)
- `pip install -e .` проходить без помилок.
- `python -c "from tools.schemas import Finding, Event, Manifest, CVE"` — ок.
- `python -c "from tools.config import load_model_config; print(load_model_config().model)"`
  друкує `qwen2.5-coder:7b`.
- Тест `tests/test_schemas.py`:
  - створює `Finding` з одним `CVE`, робить round-trip `model_dump_json` → `model_validate_json`, поля збігаються;
  - `Event.now(run_id="x", agent="recon", type="note")` має валідний `ts` типу datetime.
- `pytest -q` — зелено. Тести не ходять у мережу.
