# M0.3 — labshell: логування лаб-сесій та команд

**Контекст:** останній блок M0. Реалізує «хребет» з ARCHITECTURE.md §3–§4:
кожна лаб-сесія = прогін (run) з `manifest.json`, а кожна команда логується в
`logs/<run_id>/commands.jsonl`, щоб агенти могли «бачити» історію (KB Curator, M4).

## Ціль
Обгортка `scripts/labshell`, яка відкриває залоговану bash-сесію: створює run,
пише manifest, записує asciinema-каст (якщо є) і додає кожну виконану команду в
`commands.jsonl` (timestamp, cwd, команда, exit code).

## Файли

### 1. `tools/schemas.py` — ДОДАТИ модель (не чіпати наявні)
- `class CommandLogEntry(BaseModel)`: `ts: datetime`, `cwd: str`, `cmd: str`,
  `exit_code: int`. Серіалізується в JSON.

### 2. `scripts/new_run.py` — ініціалізація прогону
- Аргументи: `--scenario` (обов'язк.), `--targets` (через кому), `--operator`
  (default: `$USER`), `--run-id` (default: `YYYY-MM-DD_HHMMSS_<scenario>`).
- Створює `logs/<run_id>/` і `runs/<run_id>/raw/`.
- Пише `runs/<run_id>/manifest.json` через `tools.schemas.Manifest`
  (`started` = поточний UTC; `tool_versions` можна лишити `{}`).
- Друкує в stdout **лише** `run_id` (щоб bash міг захопити), решту — у stderr.

### 3. `scripts/logcmd.py` — додавання одного запису команди
- Аргументи: `--file` (шлях до commands.jsonl), `--cmd`, `--exit` (int),
  `--cwd` (default: поточний).
- Створює `CommandLogEntry` (ts = UTC now) і **дописує** один JSON-рядок у файл
  (append, newline-delimited). Створити файл/директорію за потреби.

### 4. `scripts/labshell_rc.sh` — rcfile сесії
- Спершу `source ~/.bashrc` якщо існує (щоб середовище лишалось звичним).
- Встановити логер: перед кожним промптом писати попередню команду через
  `python "$AILAB_SCRIPTS/logcmd.py" --file "$AILAB_CMDLOG" --cmd "<остання команда>"
  --exit "<її код>"`. Використати `trap ... DEBUG` для захоплення `BASH_COMMAND`
  та `PROMPT_COMMAND` для запису з реальним exit-кодом. Не логувати сам виклик логера
  (уникнути рекурсії) і порожні команди.
- Показати в промпті мітку, напр. `(lab:<run_id>)`.

### 5. `scripts/labshell` — entrypoint (bash, executable)
- Usage: `labshell <scenario> [targets] [operator]`.
- Викликає `new_run.py`, захоплює `run_id`, експортує:
  `AILAB_RUN_ID`, `AILAB_CMDLOG=logs/<run_id>/commands.jsonl`,
  `AILAB_SCRIPTS=<шлях до scripts>`.
- Запускає bash сесію з `--rcfile scripts/labshell_rc.sh`.
- Якщо в системі є `asciinema` — обгорнути сесію в
  `asciinema rec logs/<run_id>/session.cast -c "bash --rcfile ..."`;
  якщо нема — просто запустити bash з rcfile (asciinema опційна).
- `chmod +x` на `scripts/labshell`.
