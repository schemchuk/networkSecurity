# M6.3 — PDF/HTML експорт звіту

**Контекст:** продовження [ROADMAP-M6-professional.md](../ROADMAP-M6-professional.md) після
[M6.2 (професійний звіт)](M6.2-professional-report.md), уже в `main`. Зараз пайплайн віддає
лише `report.md`. Мета M6.3: той самий контент — у "товарному" вигляді (HTML для перегляду в
браузері, PDF для друку/відправки замовнику), без зміни того, що вже генерується в Markdown.

**Обмеження заліза:** без важких системних залежностей (нічого на кшталт WeasyPrint, якому
потрібні системні cairo/pango/gdk-pixbuf). Обираємо чисто pip-пакети:
- `markdown` (Markdown → HTML, розширення `tables`, бо в звіті pipe-таблиці);
- `xhtml2pdf` (HTML → PDF, чистий Python поверх reportlab, без системних бібліотек).

Додати обидва в `requirements.txt`.

## Ціль
Новий модуль `tools/report_export.py`, що бере вже згенерований Markdown-текст звіту (те, що
повертає `render_report`) і конвертує його в HTML-документ і PDF-файл. Пайплайн
(`run_recon_pipeline`) після запису `report.md` пише поруч `report.html` і `report.pdf` —
завжди, без нових CLI-прапорців (це не важка операція, тож без гейтингу).

## Файли

### `requirements.txt`
Додати рядки `markdown` і `xhtml2pdf`.

### `tools/report_export.py` — новий модуль
```python
def markdown_to_html(md_text: str, title: str = "Recon Report") -> str:
    """Render Markdown report text to a standalone styled HTML document."""

def write_html_report(md_text: str, path: str | Path, title: str = "Recon Report") -> None:
    """Render md_text to HTML and write it to path."""

def write_pdf_report(md_text: str, path: str | Path, title: str = "Recon Report") -> None:
    """Render md_text to HTML, then to PDF, and write it to path."""
```

- `markdown_to_html`: `markdown.markdown(md_text, extensions=["tables"])` для тіла; обгорнути в
  повний HTML-документ (`<!doctype html><html><head><meta charset="utf-8"><title>{title}</title>
  <style>...</style></head><body>{body}</body></html>`). Стиль — простий, читабельний, для друку:
  максимальна ширина контенту (~800px), моноширинний шрифт для таблиць необов'язковий, таблиці з
  бордерами (`border-collapse: collapse; border: 1px solid #ccc; padding: 4px 8px`), заголовки
  з невеликим відступом. Без зовнішніх ресурсів (шрифти/CDN) — усе inline у `<style>`.
- `write_html_report`: створити батьківські директорії (`Path(path).parent.mkdir(parents=True,
  exist_ok=True)`, як у `write_report`), записати результат `markdown_to_html` у `path` (UTF-8).
- `write_pdf_report`: отримати HTML через `markdown_to_html`, згенерувати PDF через
  `xhtml2pdf.pisa.CreatePDF(html_text, dest=<відкритий бінарний файл>)`. Якщо
  `pisa_status.err` truthy — підняти `RuntimeError("Failed to render PDF: ...")`. Директорії
  створювати так само, як у `write_html_report`.

### `agents/pipeline.py` — інтеграція
Після існуючого `write_report(...)`:
- прочитати щойно записаний `report.md` (або тримати markdown-текст у змінній до запису, аби не
  читати файл повторно — на розсуд, обидва варіанти ок);
- викликати `write_html_report(md_text, run_dir / "report.html", title=...)` і
  `write_pdf_report(md_text, run_dir / "report.pdf", title=...)`;
- `title` — якщо переданий `engagement` (той самий параметр з M6.2), використати
  `f"Recon Report — {engagement.client}"`, інакше `"Recon Report"`.
- `run_recon_pipeline` і далі повертає лише шлях до `report.md` (сумісність із наявними
  викликами/тестами) — HTML/PDF пишуться як побічний ефект.

## Тести

### `tests/test_report_export.py` — новий файл
- `test_markdown_to_html_contains_table_and_title`: md з заголовком `# Title` і pipe-таблицею →
  HTML містить `<table`, `<th>` або `<td>`, і `<title>Title</title>` (передати title окремим
  аргументом і перевірити, що він у виводі).
- `test_write_html_report_creates_file` (`tmp_path`): записує файл, читає назад, перевіряє
  вміст (заголовок, шматок тексту з md).
- `test_write_pdf_report_creates_valid_pdf` (`tmp_path`): записує PDF, перевіряє
  `path.read_bytes()[:5] == b"%PDF-"` і `path.stat().st_size > 0`.
- `test_write_pdf_report_invalid_html_raises` (опційно, якщо легко змоделювати помилку;
  інакше пропустити — не критично).

### `tests/test_report_and_pipeline.py` — доповнити `test_pipeline_end_to_end`
Після наявних асертів на `report_path` додати:

assert html_path.exists()
