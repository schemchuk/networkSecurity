# Specs — завдання для Kimi

Кожен файл `Mx.y-*.md` — самодостатнє завдання на **один логічний блок коду**.

## Робочий цикл
1. Claude пише спец сюди.
2. Ти копіюєш вміст спеца в термінал Kimi (VS Code).
3. Kimi реалізує код у репозиторії **і сам запускає перевірки** з розділу «Acceptance».
4. Ти кажеш Claude «M0.1 готово» — Claude дивиться `git diff`, за потреби править, робить commit + push у `main`.

## Правила для Kimi (діють для всіх спеців)
- Дотримуйся структури репозиторію з `docs/ARCHITECTURE.md`.
- Python 3, чистий та типізований код, docstring-и українською або англійською.
- Не додавай залежностей понад перелічені у спеці без потреби.
- **Перед поверненням** прожени команди з «Acceptance criteria» — усі мають бути зелені.
- Жодних мережевих викликів у тестах (лаба офлайн-first).
