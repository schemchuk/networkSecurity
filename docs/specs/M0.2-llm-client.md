# M0.2 — Ollama LLM-клієнт + smoke-тест

**Контекст:** блок 2 з M0. Спирається на `tools/config.py` (M0.1) та §5.4 ARCHITECTURE.md.
Це тонкий клієнт до локальної Ollama — фундамент, на якому агенти робитимуть LLM-кроки.

## Ціль
`tools/llm.py` — обгортка над Ollama, що вміє: (1) простий текстовий чат,
(2) структурований JSON-вивід із валідацією за Pydantic-схемою + 1 retry.

## Залежності (додати у `requirements.txt` + `pyproject.toml` dependencies)
- `ollama` (офіційний python-клієнт)

## Файли

### `tools/llm.py`
- `class LLMError(Exception)` — помилки клієнта.
- `class LLMClient`:
  - `__init__(self, config: ModelConfig | None = None)` — якщо `config is None`,
    бере `load_model_config()`. Ініціалізує `ollama.Client(host=config.host)`.
  - `chat(self, prompt: str, system: str | None = None) -> str`
    — один виклик, повертає текст. Використовує `config.model`, `config.temperature`,
      `num_ctx=config.context_tokens`.
  - `chat_json(self, prompt: str, schema: type[BaseModel], system: str | None = None,
    retries: int = 1) -> BaseModel`
    — просить модель повернути JSON, парсить і валідує через `schema.model_validate_json`.
      Використовує Ollama `format="json"`. При невдалій валідації — до `retries` повторів,
      додаючи в наступний промпт текст помилки валідації. Якщо всі спроби провалились —
      піднімає `LLMError` з останньою помилкою.
- Типізація строга, docstring-и. Жодних мережевих викликів на імпорті модуля.

### `scripts/smoke_llm.py`
- Невеликий ручний smoke-тест (запускається оператором, НЕ в pytest):
  друкує відповідь `LLMClient().chat("Say 'pong' and nothing else.")`.
- Має рядок-підказку у docstring: спершу переконатись, що `ollama serve` працює
  і модель `qwen2.5-coder:7b` завантажена.

## Тести — `tests/test_llm.py` (БЕЗ мережі!)
Не звертатись до реального Ollama. Мокати клієнт:
- `test_chat_returns_text` — замокати `ollama.Client.chat`, повернути фейкову відповідь,
  перевірити що `LLMClient.chat` віддає текст.
- `test_chat_json_valid` — мок повертає валідний JSON під тестову схему → повертається
  екземпляр схеми з правильними полями.
- `test_chat_json_retries_then_raises` — мок повертає невалідний JSON завжди →
  переконатись що зроблено `1 + retries` спроб і піднято `LLMError`.
Використовуй `unittest.mock` / `monkeypatch`.

## Acceptance criteria (Kimi ганяє перед поверненням; спершу активувати .venv)
- [ ] `pip install -e ".[dev]"` у `.venv` — ок.
- [ ] `python -c "from tools.llm import LLMClient, LLMError"` — ок.
- [ ] `pytest -q` — усі тести зелені, **жоден не ходить у мережу**.
- [ ] `python scripts/smoke_llm.py` — НЕ входить в acceptance (потребує живого Ollama);
      лишити для ручної перевірки оператором.
