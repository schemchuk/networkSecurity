# M1.1 — nmap-парсер

**Контекст:** перший блок M1 (MVP Recon Analyzer). Детермінований парсер — за принципом
§2.2 та §5.4 ARCHITECTURE.md: точний витяг даних окремо від LLM. Вихід — `list[Finding]`
за схемою з M0.1 (§7). CVE тут НЕ визначаємо (це M2).

## Ціль
`tools/nmap_parser.py`: розібрати nmap XML і повернути нормалізовані `Finding` по
кожному **відкритому** порту.

## Залежності (додати в `requirements.txt` + `pyproject.toml`)
- `python-libnmap`

## Файли

### `tools/nmap_parser.py`
- `def parse_nmap_xml(path: str | Path) -> list[Finding]`:
  - Парсить через `libnmap.parser.NmapParser.parse_fromfile`.
  - Для кожного хоста, що `up`, для кожного сервісу зі `state == "open"`:
    створити `Finding`:
    - `id`: послідовний `F-0001`, `F-0002`, … (наскрізна нумерація по всьому файлу)
    - `host`: IP/адреса хоста
    - `port`: int, `protocol`: `tcp`/`udp`
    - `service`: назва сервісу (напр. `ssh`), `product`: банер-продукт (напр. `OpenSSH`),
      `version`: версія (порожнє → `None`)
    - `source`: `"nmap"`
    - `evidence`: `f"{path}#port{port}"`
    - решта полів — дефолти (severity=info, cves=[]).
  - Закриті/фільтровані порти пропускати.
  - Якщо файл не парситься — підняти `ValueError` з пояснювальним повідомленням.
- Строга типізація, docstring-и. Без мережі, без LLM.

### `tests/fixtures/nmap_sample.xml`
Створити мінімальний, але валідний nmap XML: один хост `10.10.10.5` (up), порти:
- `22/tcp open ssh OpenSSH 8.2p1`
- `445/tcp open microsoft-ds Samba smbd 3.0.20`
- `23/tcp closed telnet` (має бути **відфільтрований**)

```xml
<?xml version="1.0"?>
<!DOCTYPE nmaprun>
<nmaprun scanner="nmap" args="nmap -sV 10.10.10.5" start="1700000000" version="7.94">
  <host starttime="1700000000" endtime="1700000010">
    <status state="up" reason="echo-reply"/>
    <address addr="10.10.10.5" addrtype="ipv4"/>
    <ports>
      <port protocol="tcp" portid="22">
        <state state="open" reason="syn-ack"/>
        <service name="ssh" product="OpenSSH" version="8.2p1" method="probed" conf="10"/>
      </port>
      <port protocol="tcp" portid="445">
        <state state="open" reason="syn-ack"/>
        <service name="microsoft-ds" product="Samba smbd" version="3.0.20" method="probed" conf="10"/>
      </port>
      <port protocol="tcp" portid="23">
        <state state="closed" reason="conn-refused"/>
        <service name="telnet" method="table" conf="3"/>
      </port>
    </ports>
  </host>
</nmaprun>
```

## Тести — `tests/test_nmap_parser.py`
- `test_parses_open_ports`: розбирає фікстуру → рівно **2** findings (23 відфільтровано).
- `test_finding_fields`: для порту 445 — `host==10.10.10.5`, `protocol=="tcp"`,
  `service=="microsoft-ds"`, `product=="Samba smbd"`, `version=="3.0.20"`,
  `source=="nmap"`, `evidence` містить `port445`.
- `test_ids_are_sequential`: id = `F-0001`, `F-0002`.
- `test_invalid_file_raises`: неіснуючий/битий файл → `ValueError`.
