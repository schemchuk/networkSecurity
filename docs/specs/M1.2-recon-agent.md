# M1.2 — BaseAgent + Recon-агент

**Контекст:** блок 2 з M1. Перший агент — за §5.2/§5.4 ARCHITECTURE.md: тонкий шар
(детерміновані дані з M1.1 + один LLM-крок збагачення). Дотримуватись меж із `docs/SCOPE.md`:
**тільки аналіз**, жодних експлойтів, жодних вигаданих CVE (CVE — це M2).

## Ціль
- `agents/base.py` — базовий клас агента: контекст прогону, LLM, append-only events.
- `agents/recon.py` — Recon-агент: збагачує кожен `Finding` (summary + next_steps)
  через LLM і пише `runs/<run_id>/findings.json`.

## Файли

### `agents/base.py`
- `class BaseAgent`:
  - `__init__(self, run_id: str, runs_dir: str | Path = "runs", llm: LLMClient | None = None)`
    - `self.run_id`, `self.run_dir = Path(runs_dir)/run_id`, створити за потреби.
    - `self.llm = llm if llm is not None else LLMClient()` (лінива — але для тестів
      передаватимемо мок; **не** створювати клієнт, якщо `llm` передано).
    - `self.name: str` — коротка назва агента (перевизначають підкласи; у базі `"base"`).
  - `emit_event(self, type, ref=None, data=None) -> None`
    - додає рядок `Event.now(run_id, agent=self.name, type, ref, data)` у
      `self.run_dir/"events.jsonl"` (append, newline-delimited JSON).
  - `run(self, *args, **kwargs)` — `raise NotImplementedError`.
- Імпорт `LLMClient` з `tools.llm`, `Event` з `tools.schemas`.

### `agents/recon.py`
- `class ReconEnrichment(BaseModel)`: `summary: str`, `next_steps: list[str] = []`.
- `RECON_SYSTEM_PROMPT: str` — системний промпт: роль recon-аналітика; тільки
  аналіз attack surface та **напрямки подальшого аналізу**; заборона пропонувати
  готові експлойти/пейлоади; не вигадувати CVE. (2–5 речень.)
- `class ReconAgent(BaseAgent)`:
  - `name = "recon"`.
  - `run(self, findings: list[Finding]) -> list[Finding]`:
    - emit `note` про старт (data: кількість findings).
    - для кожного finding:
      - побудувати промпт з фактів (host/port/service/product/version);
      - `self.llm.chat_json(prompt, schema=ReconEnrichment, system=RECON_SYSTEM_PROMPT)`;
      - записати `finding.summary` і `finding.next_steps` із результату;
      - emit `llm_call` (ref=finding.id);
      - при `LLMError` — emit `error` (ref=finding.id), лишити summary порожнім, **не падати**.
    - записати всі findings у `self.run_dir/"findings.json"` (JSON-масив, `indent=2`).
    - повернути збагачений список.

## Тести — `tests/test_recon_agent.py` (мок LLM, без мережі)
- Фейковий LLM: об'єкт із методом `chat_json(...)`, що повертає
  `ReconEnrichment(summary="...", next_steps=["..."])` (використати `unittest.mock`).
- `test_enriches_findings`: 2 findings на вході → у кожного заповнені `summary`/`next_steps`.
- `test_writes_findings_json`: `findings.json` існує, парситься назад у `list[Finding]`,
  довжина збігається.
- `test_emits_events`: `events.jsonl` містить рядки, що валідуються як `Event`
  (є `note` старту і `llm_call` на finding).
- `test_llm_error_is_tolerated`: якщо `chat_json` кидає `LLMError` — агент не падає,
  пише `error`-подію, finding лишається зі своїми базовими полями.
- Використати `tmp_path` як `runs_dir`.
